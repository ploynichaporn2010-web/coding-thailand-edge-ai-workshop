## 🎯 1. Problem & User

### a) เราจะแก้ปัญหาอะไร?
```
การแยกขวดน้ำพลาสติกและกระป๋องอะลูมิเนียมภายในโรงเรียน
เพื่อลดภาระการทำงานของพนักงานและลดการปนกันของขยะทั้ง2ประเภท

```

### b) ใครคือ user หลัก?
```
ส่งผลกระทบต่อพนักงานทำความสะอาดของโรงเรียนที่ต้องคัดแยกขยะที่
อาจจะเสี่ยงในการสัมผัสกับขยะและได้รับบาดเจ็บเมื่อโดนกระป๋องบาด

```

### c) ทำไม Edge AI ถึงเหมาะกับปัญหานี้? (ทำไมไม่ใช่ Cloud)
```
ต้อง response < 1 วินาที เพื่อเปิดช่องทิ้งขยะทันที
ติดตั้งในจุดที่ไม่มี WiFi เสถียรได้
ไม่ต้องส่งข้อมูลเสียงขึ้น Cloud → ปลอดภัยกว่า

```

## 💎 2. Value Proposition

### Unique Value (USP)
ในประโยคเดียว:

```
เครื่องแยกขยะอัตโนมัติด้วยเสียง ช่วยอาคาร/องค์กร
แยกโลหะและพลาสติกได้ทันที โดยไม่ต้องพึ่ง internet
ต่างจากถังขยะทั่วไป เพราะไม่ต้องอาศัยการตัดสินใจของคน

```

## 🚧 7. Risk & Mitigation

### Top 3 ความเสี่ยง

| ความเสี่ยง | ผลกระทบ | วิธีแก้ |
|---|---|---|
|เสียงรบกวนรอบข้าง|accuracy ต่ำ|เก็บ dataset ในสภาพแวดล้อมจริง|
|Servo ไม่ทำงานบน UNO |ไม่มี output|ทดสอบ wiring ก่อน demo|
|Dataset น้อยเกินไป |โมเดล overfit|เก็บเสียงหลายขนาด/วัสดุ|

ตัวอย่าง:
| ความเสี่ยง | ผลกระทบ | วิธีแก้ |
|---|---|---|
| Model accuracy drop ในสภาพแสงต่ำ | ใช้จริงไม่ได้ | เก็บข้อมูล low-light เพิ่มในรอบถัดไป |
| Servo ไม่ทำงานบน UNO Q | ไม่มี output | เตรียม Pixels เป็น backup |

## 💡 Tips สำหรับช่วงต่อยอด

1. **ถ้าต้องอธิบายงาน 2 นาที ทีมควรแบ่งช่วงพูดกันล่วงหน้า**
2. **ถ้าต้องโชว์สด ควรมีแผนสำรองเมื่อ internet หรือ hardware มีปัญหา**
3. **Document ทุก iteration** ใน GitHub — commit ทุก hour
4. **Token = constraint ที่สอนการตัดสินใจ** — เลือกสิ่งจำเป็น
5. **ไอเดียดี + execute ไม่จบ < ไอเดียเรียบง่าย + execute เสร็จ**
